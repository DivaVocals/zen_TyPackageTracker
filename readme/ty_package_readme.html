PURPOSE & AIM

  • This module adds the ability for your customers and admin to track packages from different carriers.
    By default the carriers included are FedEx, UPS, & USPS
  • Adds a separate sidebox that displays the customer's past orders
  • Customer can track their orders from the sidebox or from the "My Account" page
  • Configurations are done in the Admin Page. You may have up to 5 different carriers. You can change, add, or turn off carrier from this page

!!READ EVERYTHING!!

After following the installation instructions, I very strongly recommend you read through the ENTIRE "How to Use" section. There you will find out exactly what Ty Package Tracker can do.

Ready to get started? Just follow the installation instructions.

INSTALLATION

!!!!!BACKUP YOUR STORE BEFORE YOU BEGIN!!!!!

There are three key folders in the fileset

  • 1_Install_Files
  • 2_Required_Core_Edits
  • 3_Remove_From_Previous_Versions
Basic Installation/Upgrade Instructions
  1. Unzip the package contents to a temp directory
  2. Rename the YOUR_ADMIN_FOLDER folder in the 1_Install_Files folder to match the name of your shop's admin folder.. Let me repeat this as this is the one step in module installs that most OFTEN gets overlooked/missed:

    Rename the YOUR_ADMIN_FOLDER folders to match the name of your shop's admin folder
  3. Rename the YOUR_ADMIN_FOLDER folders in the 2_Modified_Core_Files folder to match the name of your store's admin folder

    Because these are core files, they have been separated in the install fileset so they are easily identified. If you have installed any mods which make changes to the files in this folder, you will need to use a tool like Winmerge or Beyond Compare to incorporate those changes into these files.
  4. Log into your store's admin before uploading the files.
    If you log into your store after uploading the files, the auto-install will still ocurr, but you will not see the install confimation message. (this is because the auto-install will take place during the login action and the confirmation message will not display during the login process)
  5. Upload ALL the files in the 1_Install_Files folder to your store directory. The directory structure within these folders is like your Zen Cart store's folder structure
  6. Upload files to from 2_Modified_Core_Files to your store directory. The directory structure within these folders is like your Zen Cart store's folder structure.

    IMPORTANT: If you have Super Orders v4.x installed see the "Integration with Other Modules" tab BEFORE uploading any of the files in the "2_Modified_Core_Files" folder.
  7. Once the files are uploaded, click ANY link in your store's admin, and the auto-installer will be triggered. If you have followed the install instructions, you will see the installation confirmation message.
  8. Go to Admin->Tools->Layout Boxes Controller to enable/disable the Previous Orders sidebox.
IMPORTANT: If you have Edit Orders or Super Orders installed see the "Integration with Other Modules" tab for instructions on using Edit Orders or Super Orders with Ty Package Tracker .
Database Modifications
  • Appends 5 fields to the existing TABLE "orders_status_history"
  • Add new table to Admin->Configuration->Ty Package Tracker (this includes a flag to turn on Ty Package Tracker fields in Edit Orders and/or Super Orders)

YOU SHOULD ALWAYS BACKUP YOUR DATABASE WHEN YOU ADD, DELETE, OR UPDATE ANY FIELDS FROM YOUR DATABASE.

  • Use "ty_package_tracker_uninstall.sql" if you want to delete all fields from this module from the database.
Uninstall Instructions

YOU SHOULD ALWAYS BACKUP YOUR DATABASE WHEN YOU ADD, DELETE, OR UPDATE ANY FIELDS FROM YOUR DATABASE.

  1. Using a tool like Winmerge or Beyond Compare you will need to carefully modify the files that were updated to install this add-on to remove the Ty Package Tracker code and replace it with the default Zen Cart Code. The files are listed in the section above.
  2. Use "ty_tracker_package_uninstall.sql" if you want to delete all fields of this module from the database. All tracking id in the database will also be deleted.

HOW TO USE

Configuring Ty Package Tracker
  1. Go to Admin->Configuration->Ty Package Tracker to configure settings
  2. Go to Admin->Customers->Orders
  3. Select the desired order
  4. Add a tracking number in the appropriate input box
  5. Add comments or change the status. (Required or your tracking information will not be saved!)*
  6. Click "Update". This will add the tracking number to the database and will also send the customer an e-mail with the tracking id and the link needed to track the package
  7. Click on the tracking ID to track packages. A separate window will display the tracking information. (Tracking ID will also be display on the Customer's My Account page)

*Note: You must add a comment or change the status prior to clicking on the "Update" button or you will get the following warning message:
"Warning: Nothing to change. The order was not updated."

Customize/Add New Carriers

To add new carriers, you will need the carriers tracking link.

Go to Admin->Configuration->Ty_Package_Tracker to configure settings.

For each new carrier, you will have to edit 2 fields - the "name" and the "tracking link".

Example:
If the URL for tracking is http://www.fedex.com/Tracking?action=track&tracknumbers=1234567890
Omit the actual tracking numbers and use http://www.fedex.com/Tracking?action=track&tracknumbers= as the tracking link.

Frequently Asked Questions
  1. I get the Warning: Nothing to change. The order was not updated.

    -> When you add a tracking number, you must either change the order status or add a comment in order to update the order with the tracking data.
  2. Is v.3.1.4 backwards compatible with older Zen Cart versions (prior to v1.5)?

    -> The short answer is no. Zen Cart v1.5 is VERY different and so is Ty Package Tracker v.3.1.4. There is no backwards compatibility support for Zen Cart versions prior to v1.5. If you are upgrading your Zen Cart v1.3.x shop to Zen Cart v1.5, DO NOT run the un-install SQL, and simply remove any previous versions of Ty Package Tracker files from your Zen Cart v1.5 store. Then install Ty Package Tracker from scratch. The good news is you will NOT need to make any database changes. The auto-installer will preserve your existing tracking data.

INTEGRATION WITH OTHER MODULES

Super Orders v4.x

(You MUST have Super Orders installed to use the Super Orders integration features)

You will need to follow the following steps to use Ty Package Tracker v.3.1.4 with Super Orders v4.x.

  • If you are running Super Orders v4.x, no patches or additional files are needed. DO NOT UPLOAD the orders.php file from the Ty Package Tracker 2_Modified_Core_Files folder.
  • If you'd like to add the shipper tracking number fields to the orders.php page, go to
    Admin > Configuration > Ty Package Tracker
    Set the "Ty Package Tracker Module Switch" to "True" to activate the Ty Package Tracker shipper tracking number fields in Super Orders.
Edit Order v4.x

(You MUST have Ty Package Tracker installed to use the Ty Package Tracker integration features)

You will need to follow the following steps to use Ty Package Tracker v.3.1.4 with Edit Orders v4.x.

  • If you are NOT running Super Orders & Edit Orders together, you will need to use a tool like Winmerge or Beyond Compare to merge the orders.php files from both Edit Orders and Ty Package Tracker.
  • If you'd like to add the shipper tracking number fields to the edit_orders.php page, go to
    Admin > Configuration > Ty Package Tracker
    Set the "Ty Package Tracker Module Switch" to "True" to activate the Ty Package Tracker shipper tracking number fields in Edit Orders.
